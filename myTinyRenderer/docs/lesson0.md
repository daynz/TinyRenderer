# 如何用500行代码实现OpenGL软件渲染

这是一个小型的软件光栅化器，如果你正在寻找一个小型的软件光线追踪器，你可以在这里找到。

我的源代码并不重要。阅读维基并实现你自己的渲染器。只有当你经历所有微小的细节时，你才会明白发生了什么。

在这篇文章系列中，我想通过编写OpenGL的克隆（一个更简化的版本）来展示OpenGL是如何工作的。令人惊讶的是，我经常遇到一些人无法克服学习OpenGL / DirectX的初始障碍。因此，我准备了一系列简短的讲座，之后我的学生们展示了相当好的渲染器。

所以，任务的表述如下：不使用第三方库（特别是图形库），得到像这样的图片：

警告：这是一份培训材料，将松散地重复OpenGL库的结构。它将是一个软件渲染器。我不想展示如何为OpenGL编写应用程序。我想展示OpenGL是如何工作的。我深信，没有理解这一点，就不可能使用3D库编写高效的应用程序。

我将尝试使最终代码大约有500行。我的学生们需要10到20小时的编程时间才能开始制作这样的渲染器。在输入时，我们得到一个带有多边形线框+纹理图片的测试文件。在输出时，我们将得到一个渲染的模型。没有图形界面，程序只是简单地生成图像。

由于目标是最小化外部依赖，我给学生们提供了一个类，允许使用TGA文件。这是最简单的格式之一，支持RGB/RGBA/黑白图像格式。所以，作为一个起点，我们将获得一种简单的方法来处理图片。你应该注意到，最初唯一可用的功能（除了加载和保存图像之外）是设置一个像素的颜色。

没有绘制线段和三角形的函数。我们将不得不全部手动完成。我提供了我与学生并行编写的源代码。但我不建议使用它，因为这没有意义。整个代码可以在github上找到，在这里你将找到我给学生的源代码。

```c
#include "tgaimage.h"
const TGAColor white = TGAColor(255, 255, 255, 255);
const TGAColor red   = TGAColor(255, 0,   0,   255);
int main(int argc, char** argv) {
    TGAImage image(100, 100, TGAImage::RGB);
    image.set(52, 41, red);
    image.flip_vertically(); // 我希望图像的原点在图像的左下角
    image.write_tga_file("output.tga");
    return 0;
}
```
